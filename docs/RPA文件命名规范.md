# RPA 文件命名规范

> 维护人：骆铭源 | 生效日期：2026-04-08
> 本文档约束 RPA 导出文件的命名格式，入库脚本按此规范做文件匹配。
> **文件名不符合规范 → 入库脚本跳过 → 当天数据缺失！**

---

## NAS 路径

- 地址：`\\192.168.1.49\数仓rpa文件\数据组文件`
- 账号：`lmy001`
- 每个平台一个文件夹，按日期建子目录：`{平台文件夹}\YYYY-MM-DD\`

## 各平台文件命名规范

### 美团

| 文件类型 | 规范文件名 | 说明 |
|---------|-----------|------|
| 商品明细 | `美团-商品明细（{门店名}）.csv` | GBK 编码，每个门店单独一个文件 |
| 账单 | `美团-账单（{门店名}）.xlsx` | 含 4 个 sheet：账单明细、订单明细、处罚&赔付、健康卡费用商家部分 |
| 推广（直营） | `美团-推广账单（直营）.xlsx` | 含 sheet：余额流水、推广费流水 |
| 推广（加盟） | `美团-推广账单（加盟）.xlsx` | 同上 |

**路径**：`\\192.168.1.49\数仓rpa文件\数据组文件\【数据组】美团数据导出\YYYY-MM-DD\`

> 华信和槐荫可以合并成"善培臣"导出（用商家ID区分门店），小猴单独导出。

**示例（2026-03-25）**：

```
【数据组】美团数据导出\2026-03-25\
├── 美团-商品明细（小猴）.csv
├── 美团-商品明细（善培臣）.csv        ← 华信+槐荫合并
├── 美团-账单（小猴）.xlsx
├── 美团-账单（善培臣）.xlsx            ← 华信+槐荫合并
├── 美团-推广账单（直营）.xlsx
└── 美团-推广账单（加盟）.xlsx
```

## 命名规则（必须遵守）

1. **花括号 `{门店名}` 替换为门店简称**（如：小猴、华信、槐荫、大坪），其余部分一字不改
2. **括号统一用中文全角括号 `（）`**，不要用半角 `()`
3. **不要在文件名中加日期、长串 ID、公司全称**
4. **门店合并规则**——华信和槐荫可合并为"善培臣"导出（用商家ID区分），其他门店不要随意合并
5. **不要改变文件格式**——CSV 就是 CSV，XLSX 就是 XLSX，不要互换
6. **目录下不要放无关文件**（如操作登记表、warn.docx 等）
