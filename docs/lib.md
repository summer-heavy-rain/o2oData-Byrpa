> 使用方法：在对话中输入 `/记录` 即可触发记录。

## 平台与数据源

**美团 / 饿了么 / 京东（财务对账通用铁律）**：
1. **先按订单号匹配**：订单明细 × 财务账单。**匹配到** → 进入公式校验；**匹配不到** → 暂存不计入当期，**但必须单独列出**（按门店汇总金额/笔数），供 xiaohou 核查。暂存订单待财务账单更新后再纳入计算并算给门店。
2. **公式校验（仅对已匹配订单）**：用已有结算公式推导 vs 平台结算金额逐单比对；**推导≠账单** → 公式未覆盖新增费用类型/列，AI 必须 **主动通知 xiaohou**（差异金额+涉及类型+建议归项），**确认后**才改公式。**禁止**静默跳过/补零/自行归项。
3. **整批缺失检测**：如果订单明细中某来源/平台有订单且时间已较靠前（≥3天），但财务账单该来源 **0 条** → 基本确定 RPA 没抓取到，AI 主动通知 xiaohou 排查对应 RPA 脚本。
详见 **`docs/京东到家商家对账单验数.md` §1.1**。

| 业务术语 | 数据来源 | 字段/计算方式 | 备注 |
|---------|---------|-------------|------|
| 美团结算金额 | `ods_rpa_meituan_fin_order` | `商家应收款`（只取交易类型=外卖订单） | 推广充值不算，那是充值不是消耗 |
| 饿了么结算金额 | 对账：**`SUM(订单应收_col_13)`** = 公式还原；订单 **`商户应收金额`**（已取消/全退 **正→0、负保留**）。若要对齐平台 **`SUM(结算金额_col_11)`**：再加 **`订单类型` 含「赔偿」行的 `结算金额_col_11` 合计**（赔偿行订单应收为 0） | 与「净营业额合计」不是同一口径 | 详见 **`docs/饿了么结算字段对照.md`** |
| 京东结算金额 | `ods_rpa_jd_finance` + **`ods_rpa_jd_order."订单来源"`** | **小时达**：长表 **`SUM(结算金额)`** + `费用类型` 枚举（`工作簿5.xlsx` 思路）。**到家**：平台 **「商家对账单」宽表**，**应结金额** 与列代数和恒等式见 **`docs/京东到家商家对账单验数.md`**；**开票金额**不入应结公式，**阶梯扣点佣金**并入**平台扣点佣金**；数仓长表与到家宽表尚未合一入仓时分别对账 |

## 退款处理（三平台通用规则）

**退款类交易行必须纳入结算计算**，不能只取正向订单。

**延迟退款归类**：财务账单中有退款行，但退款订单号不在当日订单明细中 → 说明是之前下单、滞后退款。**单独归类为「延迟退款金额」**，在门店结算中独立一行扣减，方便核查。延迟退款仍算给门店（扣减），只是单独列出来避免与当日经营数据混淆。

| 平台 | 正向订单 | 退款处理 |
|------|---------|---------|
| 美团 | `交易类型=外卖订单` | `订单退款` / `订单部分退款` / `部分退款冲抵` **纳入**，按订单号匹配当日明细，匹配不到的归延迟退款 |
| 饿了么 | 财务 `订单应收_col_13` | 已取消/全退：**商户应收 >0 → 对账 0**；**<0 → 保留负值**（与 `o2o_eleme_order_fin_settlement_compare.sql` 一致）；赔偿行单独处理 |
| 京东 | 小时达=长表 `SUM(结算金额)` / 到家=宽表 `应结金额` | 退款类行纳入，按订单号匹配当日明细，匹配不到的归延迟退款 |

## 美团还原公式

```
商家应收款 = 商品总价 + 用户配送费 + 餐盒费 + 打包袋
           + 商家活动支出(负)
           + 佣金(负)
           + 配送服务费(负)
           + 配送费返利
           + 公益捐款(负)
```

- **平台补贴、代理商补贴**不包含在商家应收款内（平台给用户的，不影响商家结算）
- 已验证 2146/2146 笔对齐

## 饿了么还原公式

**对账口径（= 公式还原）= 订单应收**：财务 `订单应收_col_13` 按单号 SUM；订单明细侧 **`商户应收金额`**（规则同上）。  
**平台「结算金额」列合计**：`SUM(结算金额_col_11) = SUM(订单应收_col_13) + SUM(结算金额_col_11 WHERE 订单类型含「赔偿」)`（赔偿行订单应收为 0）。

**11 项行级恒等式**（DB 值已带正负号，全部直接相加；2026-03-28 最终验证 3.25 全量 1370 笔 0 差异）：

```
结算金额 = 商品金额(col_17) + 应收运费(物流净营业额)
         + 实收佣金(col_55) + 实收支付服务费(col_59)
         + 活动补贴(col_20) + 代金券补贴(col_21) + 商家配送费补贴总额(col_28)
         + 履约技术服务费(平台履约技术服务费) + 物流配送费(物流网络配送费)
         + 骑手小费(其他支出_col_83) + 优惠金额合计(其他支出_col_81)
```

- 等价于：`净营业额合计 + 服务费用合计 + 其他支出`（行级恒等）
- **col_55 = 实收佣金**，**col_59 = 实收支付服务费**（`平台交易技术服务费` 是两者汇总，不能同时入公式）
- **col_56 / col_60 = 计费基数**（用户支付金额），不入公式
- **骑手小费在 `其他支出_col_83`**（非 `服务费用合计_col_74`，后者120天全0），仅超客配送订单有值
- **活动补贴/代金券取净营业额子项**（col_20/col_21），**非**资金来源维度（资金来源会导致双重计入）
- 详见 `docs/饿了么结算字段对照.md` 完整列映射表

## 京东还原公式

**前置**：先看 **`订单来源`**（`ods_rpa_jd_order`）——**小时达** 与 **到家** 两套形态不同，**禁止混用**。

**小时达（长表）**：对账锚 **`SUM(结算金额)`** 按 `到家业务单号`（`ods_rpa_jd_finance` 行全加，符号已在列中）。**包装费（独立分项）**：`费用类型` ∈ {**`到家包装费`**, **`包装费`**}（字面以导出为准）**单独**做 **`SUM(结算金额)`**，**不计入**「商家补贴」分项（**账单未出该类型则当期该项不算**，见 `京东到家商家对账单验数.md` §1.1）。

**到家（商家对账单宽表）**：对账锚 **`应结金额`**；列名与恒等式见 **`docs/京东到家商家对账单验数.md`**。口述「商家承担消费」在导出中为 **`商家承担小费`**，须按表头修正映射。

**经营侧分项（小时达，`费用类型` 白名单）**——在样本「活动 / 商家补贴 / 财务运费 / 平台扣点」基础上，**包装费单独成项**（**不**并入商家补贴）：

| 样本列名 | 小时达 `费用类型` 归入（枚举见文档） |
|----------|----------------------------------------|
| 活动补贴 | 平台券/京东平台券/京豆/积分等 |
| 商家补贴 | 商家券、账扣款、限时抢购商家等（**不含** `到家包装费` / 包装类 **`包装费`** 行） |
| **包装费** | **`到家包装费`**、与导出一致的 **`包装费`**（单独 `SUM(结算金额)`） |
| 财务运费金额 | 基础配送、远距离、满减运费、秒送基础服务费、免运券等 |
| 平台扣点佣金 | 代收到家佣金、CPS 推广服务费等 |

**到家宽表**（与小时达分列）：货款=`订单原价`，活动=`平台承担补贴(市场费)`，运费相关=货补/运补/自送/小费/取件费等列之和，**仅此处**宽表 **`包装费`** 列（表头有则纳入应结），扣点=`总佣金(…)(可开票)`+`基础服务费`+**`阶梯扣点佣金`**；**`开票金额`**不入应结公式；**应结**=上列代数和（详见 **`docs/京东到家商家对账单验数.md`**）。`ods_rpa_jd_finance` 暂无此宽表结构时，以导出 xlsx 或专用入仓表验数。

**补贴类**行（部分类型）：`结算金额` = `京东承担金额`，`商家承担金额`=0；其余以 **`结算金额` 为准**。

**辅助校验（小时达长表）**：活动补贴白名单行 `结算金额`≈`京东承担金额`；脚本 **`sql/o2o_jd_xiaoshida_subsidy_jd_carry_validate.sql`**。

**辅助校验（到家宽表）**：**`scripts/validate_jd_daojia_merchant_statement.py`** 对「商家对账单」xlsx 跑应结恒等式。
