# O2O 日报 — 怎么改经营指标

> 写给数据专员看的，不需要编程基础。

---

## 一、你可以改什么

项目里有 **4 个指标计算文件**，每个文件只负责一件事，20-40 行，打开就能看懂：

```
dbt/models/intermediate/
├── int_item_cost.sql          ← 成本怎么算
├── int_item_tickets.sql       ← 票数怎么算
├── int_maiyatian_shipping.sql ← 麦芽田运费怎么算
└── int_promotion_cost.sql     ← 推广费怎么算
```

你还可以改：
- `sql/` 下面的即席查询（直接在数据库里跑的 SQL）

**其他文件不要动。** 特别是 `staging/` 和 `marts/` 开头的文件。

---

## 二、改之前先做什么

### 用 AI 来帮你改

打开你要改的文件（比如 `int_item_tickets.sql`），告诉 AI：

> "我要修改票数的计算逻辑，退款订单的票数应该为0，帮我改这个文件"

AI 会帮你改。**但你要检查改完的数字对不对。**

---

## 三、改完怎么验证

### 跑一下 dbt（生成新的报表数据）

打开终端，执行：

```bash
cd dbt
dbt run --profiles-dir .
```

看到 `Completed successfully` 就是成功了。如果报错，把错误信息发给骆铭源。

### 查看结果对不对

在数据库里跑这条 SQL：

```sql
SELECT
  SUM(revenue)    AS 收入,
  SUM(cost)       AS 成本,
  SUM(item_count) AS 票数,
  SUM(net_profit)  AS 净利润
FROM ads_o2o_daily_report
WHERE dt = '2026-03-25';
```

跟校验表对比，差距在 5% 以内算合理。

---

## 四、改完怎么提交

```bash
git checkout -b feature/我改了什么
git add .
git commit -m "修改: 改了什么什么"
git push origin feature/我改了什么
```

然后告诉骆铭源"我推了一个分支，帮我看下"。**不要直接 push 到 main。**

## 五、4 个指标文件的说明

### int_item_cost.sql — 成本

> 每一单卖出去的东西值多少钱（进货成本）

算法：从 mini橙 的订单明细里取每个商品编码，去「编码匹配表」里找对应的总部招商价，乘以数量和换算比。

### int_item_tickets.sql — 票数

> 今天一共卖了多少个商品行项目

算法：mini橙 订单明细表里，每一行（每个 SKU）算 1 票。取消的订单不算。

### int_maiyatian_shipping.sql — 麦芽田运费

> 第三方配送产生的运费

算法：麦芽田报表里，按门店求和总配送费。已取消的单不算。

### int_promotion_cost.sql — 推广费

> 在各平台投广告花了多少钱

算法：美团从「推广美团自营」表取，饿了么从推广数据表取。京东暂时没有。

## 六、遇到问题怎么办

| 情况 | 怎么做 |
|------|--------|
| dbt run 报错了 | 截图发给骆铭源 |
| 不知道某个字段是什么意思 | 看 `docs/FTP_TO_ODS_MAPPING.md`，里面写了每个文件对应哪张表 |
| 想新加一个指标（比如企客返利） | 你可以自己在 `intermediate/` 下新建文件（比如 `int_qike_rebate.sql`），写好计算逻辑，跑 `dbt run` 确认不报错，然后告诉骆铭源"帮我接到最终报表里"。他会在 `ads_o2o_daily_report.sql` 里加一行 JOIN 把你的指标接进去 |
| 数字跟校验表差太多 | 先别提交，截图对比发给骆铭源一起看 |
| git 操作不会 | 问 AI "帮我创建一个分支并提交" |
